ITERACIONES

  • Iteración1: Custom Web Components sencillos.

    • Arriba se muestra un componente web sin ninguna interacción, únicamente un label que se carga desde un componente.
    • Abajo hay otro componente web; esta vez un texto junto a un botón donde existe interacción. Al pulsar el botón se actualiza el valor de la etiqueta; aunque estar ambos elementos dentro del mismo componente web no existe comunicación entre diferentes componentes, únicamente dentro del mismo.
      Además se hace uso de un atributo HTML para indicar el valor inicial a 0 en el texto.

  • Iteración2: Comunicación entre Custom Web Components y uso de API y gRPC para obtener datos.

    • A la izquierda existen tres componentes personalizados donde existe comunicación entre ellos. El número central se actualiza según los valores que reciba de los inputs laterales. Esta comunicación se realiza mediante eventos donde los componentes "hermanos" no se conocen entre ellos.
    • En la parte derecha se utiliza una API para obtener una lista. Cada botón es un componente diferente y al recibir los datos, ambos se muestran en un tercer componente.
    • Es importante tener en cuenta que para ejecutar estos componentes es necesario una preparación previa (desplegar servidor que devuelva información, levantar proxy para el gRPC, etc...) por lo que todavía no son muy indepentientes y el proyecto sigue siendo compacto.

  • Iteración3: Creación de una página web utilizando múltiples web components. Se definen los diferentes web components a utilizar (se muestran divididos en recuadros de diferentes colores).

    • De aquí en adelante se intenta codificar otro proyecto web realizado anteriormente pero haciendo uso de componentes web personalizados. Para ello se ha decidido una separación de la interfaz en componentes que se muestra en esta iteración así como la comunicación entre ellos.
    • En esta iteración todavía no existe ninguna carga de datos ni ningún estilo en la página.

  • Iteración4: Uso de APIs (servidor Express y servidor gRPC) para obtener datos externos.

    • De los cinco componentes en los que se ha dividido la interfaz, 4 de ellos necesitan de llamadas para obtener la información, por lo que se ha decidido utilizar un servidor en Express para dos de ellos y un servidor gRPC para los otros dos.
      • Express: Se utilizará para el web component de la derecha country-info-wc (información de un país) y el inferior country-statics-wc (estadísticas de un país).
      • gRPC: Se utilizará para obtener los datos a mostrar en el mapa map-wc y los datos de los países en el componente superior country-data-wc.

  • Iteración5:Se renderizan las vistas de los anteriores ejemplos para poder ejecutarlo desde un servidor y más adelante incluir las aplicaciones en un contenedor. Para esa iteración se realizarán tres sub-iteraciones que incluirán renderizar en el servidor cada uno de los proyectos anteriores. ¡Los siguientes ejemplos solo están disponibles si se han levantado los servidores correspondientes!
